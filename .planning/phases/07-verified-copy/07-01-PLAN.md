  <name>Task 1: Add copy_operations table to schema</name>
  <files>src/drivecatalog/schema.sql</files>
  <action>
Add copy_operations table to track file copy operations:
- id: INTEGER PRIMARY KEY AUTOINCREMENT
- source_file_id: INTEGER REFERENCES files(id) - the source file
- dest_drive_id: INTEGER REFERENCES drives(id) - destination drive
- dest_path: TEXT NOT NULL - relative path on destination
- source_hash: TEXT NOT NULL - SHA256 of source before copy
- dest_hash: TEXT NOT NULL - SHA256 of destination after copy
- verified: INTEGER NOT NULL - 1 if hashes match, 0 if mismatch
- bytes_copied: INTEGER NOT NULL - size in bytes
- started_at: TEXT NOT NULL - timestamp when copy started
- completed_at: TEXT NOT NULL - timestamp when copy finished

Add index on source_file_id for querying copy history of a file.

<task type="auto">
  <name>Task 2: Create copier module with streaming SHA256</name>
  <files>src/drivecatalog/copier.py</files>
  <action>
Create copier.py module with:

1. CHUNK_SIZE constant (64KB for streaming reads)

2. `copy_file_verified(source: Path, dest: Path, progress_callback: Callable | None = None) -> CopyResult`
   - Returns dataclass CopyResult with: source_hash, dest_hash, verified (bool), bytes_copied, error (str | None)
   - Stream-read source file in chunks, computing SHA256 while writing to dest
   - After copy complete, re-read dest file computing SHA256
   - Compare hashes, set verified = True if match
   - Call progress_callback(bytes_written) after each chunk if provided
   - On any error (OSError, PermissionError), return CopyResult with error message
   - Create parent directories of dest if they don't exist (Path.mkdir(parents=True, exist_ok=True))

3. `log_copy_operation(conn, source_file_id: int, dest_drive_id: int, dest_path: str, result: CopyResult) -> int`
   - Insert row into copy_operations table
   - Return the inserted row id

Use hashlib.sha256() for hashing. Do NOT use xxhash (xxhash is for fast duplicate detection, SHA256 is for integrity verification).
  </action>
  <verify>Run `python -c "from drivecatalog.copier import copy_file_verified, CopyResult"` - imports without error</verify>
  <done>copier.py exists with copy_file_verified function that returns CopyResult with hash verification</done>
</task>

<task type="auto">
  <name>Task 3: Add drives copy CLI command</name>
  <files>src/drivecatalog/cli.py</files>
  <action>
Add `@drives.command()` for copy operation:

```
@drives.command()
@click.argument("source_drive")
@click.argument("source_path")
@click.argument("dest_drive")
@click.option("--dest-path", "-d", help="Destination path (defaults to same as source)")
def copy(source_drive: str, source_path: str, dest_drive: str, dest_path: str | None) -> None:
```

Implementation:
1. Look up source_drive and dest_drive by name (error if not found)
2. Validate source_drive is mounted and source_path exists
3. Validate dest_drive is mounted
4. Look up source file in database by drive_id and path (error if not in catalog - user should scan first)
5. Determine dest_path: use --dest-path if provided, else same relative path as source
6. Check dest file doesn't already exist (error with "use --force to overwrite" hint, but don't implement --force yet)
7. Call copy_file_verified with progress callback that updates Rich progress bar
8. Log operation via log_copy_operation
9. Display result table: source, dest, bytes, source_hash (truncated), dest_hash (truncated), verified status
10. If verified: print_success("Copy verified successfully")
11. If not verified: print_error("VERIFICATION FAILED - hashes do not match!")

Progress display: Use get_progress() with task showing "Copying [filename]..." and total=file_size_bytes.

Import copy_file_verified, log_copy_operation, CopyResult from drivecatalog.copier.
  </action>
  <verify>Run `drivecatalog drives copy --help` - shows usage with SOURCE_DRIVE, SOURCE_PATH, DEST_DRIVE arguments</verify>
  <done>drives copy command works with progress display and verification output</done>
