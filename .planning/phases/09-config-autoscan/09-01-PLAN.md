  <action>
Create config.py module with:
1. Config dataclass with fields: auto_scan_enabled (bool, default True), auto_scan_drives (list[str] | None, default None = all drives)
2. get_config_path() returning ~/.drivecatalog/config.yaml using same pattern as get_db_path()
3. load_config() that reads YAML, returns Config (creates default if missing)
4. save_config(config: Config) that writes YAML

Add pyyaml dependency to pyproject.toml.

Use ruamel.yaml or pyyaml for YAML handling. Prefer pyyaml for simplicity.

<task type="auto">
  <name>Task 2: Integrate auto-scan with watcher</name>
  <files>src/drivecatalog/watcher.py, src/drivecatalog/cli.py</files>
  <action>
Modify watcher to auto-scan on mount:

1. In watcher.py, add auto_scan_on_mount(mount_path: Path, db_path: Path) function:
   - Load config via load_config()
   - If auto_scan_enabled is False, return early
   - If auto_scan_drives is set and volume name not in list, return early
   - Look up drive by mount_path in database using get_drive_by_mount_path() (add this helper to drives.py)
   - If drive not registered, skip (only auto-scan registered drives)
   - Call scan_drive() for the drive
   - Log result via Rich console

2. In cli.py watch command, update on_mount callback:
   - After printing mount status, call auto_scan_on_mount()
   - Use threading.Thread for non-blocking scan (scan can take time)

The watcher should continue monitoring during scans. Use daemon=True thread.
  </action>
  <verify>
Register a drive, start `drivecatalog drives watch`, mount the drive (or simulate by inserting USB).
Verify scan starts automatically and watcher continues responding to events.
  </verify>
  <done>Mounted registered drives auto-scan when config.auto_scan_enabled is True, watcher stays responsive</done>
</task>
