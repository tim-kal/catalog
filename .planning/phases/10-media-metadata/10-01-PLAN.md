**Schema changes:**
Add media_metadata table to schema.sql after copy_operations table:

```sql
-- Media metadata table (extracted via ffprobe)
CREATE TABLE IF NOT EXISTS media_metadata (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    file_id INTEGER NOT NULL UNIQUE REFERENCES files(id) ON DELETE CASCADE,
    duration_seconds REAL,
    codec_name TEXT,
    width INTEGER,
    height INTEGER,
    frame_rate TEXT,
    bit_rate INTEGER,
    extracted_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX IF NOT EXISTS idx_media_metadata_file_id ON media_metadata(file_id);
```

**media.py module:**
Create src/drivecatalog/media.py with:

1. `MEDIA_EXTENSIONS` set containing video formats:
   - Common: .mp4, .mov, .mkv, .avi, .wmv, .webm, .m4v
   - Professional: .mxf, .r3d, .braw, .ari, .prores

2. `is_media_file(path: Path) -> bool` - checks extension against MEDIA_EXTENSIONS

3. `MediaMetadata` dataclass:
   - duration_seconds: float | None
   - codec_name: str | None
   - width: int | None
   - height: int | None
   - frame_rate: str | None (stored as fraction like "24000/1001")
   - bit_rate: int | None

4. `extract_metadata(file_path: Path) -> MediaMetadata | None`:
   - Run: `ffprobe -v quiet -print_format json -show_streams -show_format <file>`
   - Parse JSON output, find first video stream
   - Extract from video stream: codec_name, width, height, r_frame_rate
   - Extract duration from format (fallback to stream duration)
   - Extract bit_rate from format
   - Return None on error (subprocess failure, file not found, no video stream)
   - Use subprocess.run with capture_output=True, timeout=30

**Avoid:** Don't use ffmpeg-python library (adds complexity). Direct subprocess is simpler and more reliable. Don't raise exceptions - return None for graceful degradation.

**CLI command:**
Add `@drives.command()` for `media` subcommand:

```python
@drives.command()
@click.argument("name")
@click.option("--force", is_flag=True, help="Re-extract metadata for all media files")
def media(name: str, force: bool) -> None:
    """Extract metadata for video files on a drive.

    NAME is the registered name of the drive.
    Requires ffprobe to be installed (`brew install ffmpeg`).
    """
```

Implementation:
1. Look up drive by name (same pattern as hash command)
2. Query files: if force, all files with is_media=1 OR extension in MEDIA_EXTENSIONS; else files with extensions matching MEDIA_EXTENSIONS that don't have media_metadata entries
3. For each file with media extension:
   - Update files.is_media = 1 if not already set
   - Call extract_metadata() to get metadata
   - Insert/update media_metadata table
   - Show Rich progress with file name and count
4. Print summary table: files processed, metadata extracted, errors (ffprobe failures)

**Database update:**
Ensure init_db runs CREATE TABLE IF NOT EXISTS for media_metadata (schema.sql already handles this via executescript).

Add import in cli.py: `from drivecatalog.media import MEDIA_EXTENSIONS, extract_metadata`

**CLI patterns to follow:**
- Use same connection management as hash command
- Use get_progress() context manager for progress bar
- Print summary table using Rich Table
- Use print_success/print_error for status messages

**Avoid:** Don't spawn ffprobe in parallel (disk I/O bottleneck on external drives). Process sequentially.
