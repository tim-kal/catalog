**Prior decisions:**
- Use ffprobe subprocess (not ffmpeg-python library) - established in Phase 10
- Return None/empty on errors for graceful degradation - established in Phase 10
- Store extracted_at timestamps for tracking - established in Phase 10

**Technical approach:**
- Use `-v error` instead of `-v quiet` to capture corruption messages to stderr
- Parse stderr for error messages (non-empty stderr = integrity issues)
- Store integrity status and error details in media_metadata table (extend existing table)
</context>

  <action>
1. In schema.sql, add to media_metadata table:
   - integrity_verified_at TEXT (timestamp of last verification)
   - integrity_errors TEXT (error messages from ffprobe, NULL if clean)

2. In media.py, add IntegrityResult dataclass:
   - is_valid: bool
   - errors: list[str] (empty if valid)

3. In media.py, add check_integrity(file_path: Path) -> IntegrityResult | None:
   - Run ffprobe with `-v error -i {file} -f null -` (decodes container structure only)
   - Capture stderr
   - If stderr empty: valid file
   - If stderr non-empty: parse error lines into list
   - Return None on exceptions (graceful degradation like extract_metadata)
   - Use 60 second timeout (longer than metadata extraction for large files)
  </action>
