<task type="auto">
  <name>Task 1: Create DriveDetailView with status display</name>
  <files>DriveCatalog/Views/Drives/DriveDetailView.swift</files>
  <action>
Create DriveDetailView that displays detailed information about a selected drive.

Props:
- let drive: DriveResponse (passed from list selection)

State:
- @State status: DriveStatusResponse? = nil
- @State isLoadingStatus = true
- @State activeOperation: OperationResponse? = nil

View body - use Form or GroupBox sections:

**Drive Info Section:**
- Name (bold, larger)
- Mount path (monospace font)
- Total size (formatted: "500 GB")
- UUID (if available, smaller gray text)

**Status Section:**
- Mounted indicator: Circle.fill with green (mounted) or red (unmounted) + text
- File count: "1,234 files catalogued"
- Hash coverage: Progress bar + percentage (e.g., "78.5% hashed")
- Media files: "45 media files"
- Last scanned: Relative date or "Never scanned"

Load status in .task {} using APIService.shared.fetchDriveStatus(name: drive.name)
Add refresh button to reload status.

Use ByteCountFormatter for size display.
Use RelativeDateTimeFormatter for last scanned date.
  </action>
  <verify>xcodebuild build succeeds</verify>
  <done>DriveDetailView displays drive info and status with proper formatting</done>
</task>

  <name>Task 2: Add action buttons with operation polling</name>
  <files>DriveCatalog/Views/Drives/DriveDetailView.swift, DriveCatalog/Views/Drives/DriveListView.swift</files>
  <action>
Add action buttons to DriveDetailView:

**Actions Section:**
- "Scan Drive" button - calls triggerScan(), disabled if not mounted
- "Compute Hashes" button - calls triggerHash(), disabled if not mounted

When operation triggered:
- Store operation ID in state
- Start polling with Timer or Task.sleep loop (every 2 seconds)
- Show ProgressView with operation status while running
- When complete, refresh status and clear operation

Button states:
- Disabled during active operation
- Show spinner on the active button
- Show success/failure briefly after completion

Wire up navigation:
- In DriveListView, add NavigationLink to DriveDetailView for each row
- Use NavigationStack or NavigationSplitView detail pattern

The NavigationSplitView in ContentView.swift already handles sidebar selection.
DriveListView should handle its own internal navigation to detail OR
use the existing selection binding pattern from ContentView.
