**`get_drive_file_distribution(conn: Connection) -> list[dict]`**

For each drive, compute:
- `drive_id`, `drive_name`
- `total_files`: count of all files on this drive
- `total_size_bytes`: sum of all file sizes on this drive
- `unique_files`: count of files whose `partial_hash` exists ONLY on this drive (no other drive has that hash). Include unhashed files as unique (they can't be confirmed as duplicated).
- `unique_size_bytes`: total size of unique files
- `duplicated_files`: count of files whose `partial_hash` exists on at least one other drive
- `duplicated_size_bytes`: total size of duplicated files
- `reclaimable_bytes`: size of files that are safely deletable (duplicated across drives, so copies exist elsewhere). This equals `duplicated_size_bytes`.
- `total_bytes` (drive capacity from drives table), `used_bytes`, `free_bytes` (computed: total_bytes - used_bytes, or None if either is None)

Use a single efficient SQL query with a CTE (like the `hash_stats` pattern in duplicates.py) that joins files with a subquery counting `COUNT(DISTINCT drive_id)` per `partial_hash`. Classify files where `drive_count = 1` as unique, `drive_count > 1` as duplicated.

Follow the exact pattern from `duplicates.py`: accept `conn: Connection` parameter, use `conn.execute()`, return plain dicts.

**`get_consolidation_candidates(conn: Connection) -> list[dict]`**

A drive is a "consolidation candidate" if ALL its unique files (hash only on that drive + unhashed files) can fit on other connected drives that have sufficient free space.

For each drive, determine:
1. `unique_size_bytes` from the distribution analysis
2. Available target drives: other drives where `total_bytes` and `used_bytes` are both not None and `(total_bytes - used_bytes) > 0`
3. `total_available_space`: sum of free space across all target drives
4. `is_candidate`: True if `unique_size_bytes <= total_available_space`
5. `target_drives`: list of dicts with `drive_name`, `free_bytes` for each drive with free space, sorted by free_bytes descending

Return list of dicts, one per drive, each containing the drive's distribution stats plus `is_candidate`, `target_drives`, `total_available_space`.

**Important edge cases:**
- Drives with `total_bytes = None` or `used_bytes = None` cannot be target drives (we don't know their capacity) but CAN be source candidates (we know their unique file sizes from the catalog)
- A drive should not appear in its own target list
- If a drive has 0 unique files, it's trivially a candidate (nothing to move)

**`get_consolidation_strategy(conn: Connection, source_drive_name: str) -> dict`**

Given a source drive, produce an optimal plan for moving its unique files to other drives, minimizing total bytes transferred.

Algorithm:
1. Look up the source drive by name. Raise ValueError if not found.
2. Get all unique files on the source drive (files whose `partial_hash` appears on no other drive, PLUS unhashed files). Query: select files from the source drive where either `partial_hash IS NULL` OR the hash's `COUNT(DISTINCT drive_id) = 1`.
3. Get available target drives (other drives with known capacity and free space > 0). For each: `drive_name`, `free_bytes` (total_bytes - used_bytes).
4. Sort unique files by size descending (place large files first — greedy bin-packing).
5. Sort target drives by free space descending.
6. Greedy assignment: for each unique file, assign it to the target drive with the most remaining free space that can fit it. Update that target's remaining capacity. If no target can fit a file, add it to an `unplaceable` list.
7. Return:
   ```python
   {
       "source_drive": source_drive_name,
       "total_unique_files": int,
       "total_unique_bytes": int,
       "total_bytes_to_transfer": int,  # same as total_unique_bytes minus unplaceable
       "is_feasible": bool,  # True if unplaceable is empty
       "assignments": [
           {
               "target_drive": str,
               "file_count": int,
               "total_bytes": int,
               "files": [{"path": str, "size_bytes": int, "partial_hash": str | None}],
           }
       ],
       "unplaceable": [{"path": str, "size_bytes": int, "partial_hash": str | None}],
       "target_drives": [
           {"drive_name": str, "capacity_bytes": int, "free_before": int, "free_after": int}
       ],
   }
   ```

**Optimization detail:** The greedy "largest-first, most-free-space-first" approach is a well-known approximation for bin packing that works well in practice. After assigning each file, re-sort targets by remaining free space so subsequent files go to the currently-most-available target.

**Edge case handling:**
- Source drive with 0 unique files: return immediately with empty assignments, is_feasible=True
- No available target drives: all unique files go to unplaceable, is_feasible=False
- Source drive not found: raise ValueError with clear message
