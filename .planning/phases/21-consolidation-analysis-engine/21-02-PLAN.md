**Models to create:**

1. `DriveDistribution(BaseModel)` — per-drive file distribution stats:
   - `drive_id: int`
   - `drive_name: str`
   - `total_files: int`
   - `total_size_bytes: int`
   - `unique_files: int`
   - `unique_size_bytes: int`
   - `duplicated_files: int`
   - `duplicated_size_bytes: int`
   - `reclaimable_bytes: int`
   - `total_bytes: int | None` (drive capacity)
   - `used_bytes: int | None`
   - `free_bytes: int | None`

2. `DriveDistributionResponse(BaseModel)` — wraps the list:
   - `drives: list[DriveDistribution]`
   - `total_drives: int`

3. `TargetDrive(BaseModel)` — a potential target for consolidation:
   - `drive_name: str`
   - `free_bytes: int`

4. `ConsolidationCandidate(BaseModel)` — per-drive candidacy info:
   - `drive_id: int`
   - `drive_name: str`
   - `total_files: int`
   - `total_size_bytes: int`
   - `unique_files: int`
   - `unique_size_bytes: int`
   - `duplicated_files: int`
   - `duplicated_size_bytes: int`
   - `reclaimable_bytes: int`
   - `is_candidate: bool`
   - `total_available_space: int`
   - `target_drives: list[TargetDrive]`

5. `ConsolidationCandidatesResponse(BaseModel)`:
   - `candidates: list[ConsolidationCandidate]`
   - `total_drives: int`
   - `consolidatable_count: int` (count where is_candidate=True)

6. `StrategyFile(BaseModel)` — a file in the strategy:
   - `path: str`
   - `size_bytes: int`
   - `partial_hash: str | None`

7. `StrategyAssignment(BaseModel)` — files assigned to a target:
   - `target_drive: str`
   - `file_count: int`
   - `total_bytes: int`
   - `files: list[StrategyFile]`

8. `StrategyTargetDrive(BaseModel)` — target capacity impact:
   - `drive_name: str`
   - `capacity_bytes: int`
   - `free_before: int`
   - `free_after: int`

9. `ConsolidationStrategyResponse(BaseModel)`:
   - `source_drive: str`
   - `total_unique_files: int`
   - `total_unique_bytes: int`
   - `total_bytes_to_transfer: int`
   - `is_feasible: bool`
   - `assignments: list[StrategyAssignment]`
   - `unplaceable: list[StrategyFile]`
   - `target_drives: list[StrategyTargetDrive]`

**Create `src/drivecatalog/api/routes/consolidation.py`:**

Follow the exact pattern from `routes/duplicates.py`: import router from fastapi, import models, import engine functions, use `get_connection()` with try/finally.

```python
router = APIRouter(prefix="/consolidation", tags=["consolidation"])
```

Three endpoints:

1. **`GET /consolidation/distribution`** -> `DriveDistributionResponse`
   - Calls `get_drive_file_distribution(conn)`
   - Maps each dict to `DriveDistribution` model
   - Returns wrapped in `DriveDistributionResponse` with total_drives count

2. **`GET /consolidation/candidates`** -> `ConsolidationCandidatesResponse`
   - Calls `get_consolidation_candidates(conn)`
   - Maps each dict to `ConsolidationCandidate` model (including nested `TargetDrive` list)
   - Returns wrapped in `ConsolidationCandidatesResponse` with total_drives and consolidatable_count

3. **`GET /consolidation/strategy`** -> `ConsolidationStrategyResponse`
   - Query parameter: `drive: str = Query(..., description="Source drive name")`
   - Calls `get_consolidation_strategy(conn, drive)`
   - Catches `ValueError` and raises `HTTPException(404, detail=str(e))`
   - Maps the returned dict to `ConsolidationStrategyResponse` (including nested `StrategyAssignment`, `StrategyFile`, `StrategyTargetDrive`)

Each endpoint follows the `conn = get_connection()` / `try: ... finally: conn.close()` pattern used throughout the existing routes.
