Add two new tables to schema.sql after the existing folder_stats table:

**migration_plans table:**
```sql
CREATE TABLE IF NOT EXISTS migration_plans (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source_drive_id INTEGER NOT NULL REFERENCES drives(id),
    source_drive_name TEXT NOT NULL,
    status TEXT NOT NULL DEFAULT 'draft',  -- draft, validated, executing, completed, failed, cancelled
    total_files INTEGER NOT NULL DEFAULT 0,
    files_to_copy INTEGER NOT NULL DEFAULT 0,
    files_to_delete INTEGER NOT NULL DEFAULT 0,  -- already backed up, just need source deleted
    total_bytes_to_transfer INTEGER NOT NULL DEFAULT 0,
    files_completed INTEGER NOT NULL DEFAULT 0,
    bytes_transferred INTEGER NOT NULL DEFAULT 0,
    files_failed INTEGER NOT NULL DEFAULT 0,
    errors TEXT,  -- JSON array of error strings
    operation_id TEXT,  -- links to in-memory operation tracker
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    started_at TEXT,
    completed_at TEXT
);
```

**migration_files table:**
```sql
CREATE TABLE IF NOT EXISTS migration_files (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    plan_id INTEGER NOT NULL REFERENCES migration_plans(id) ON DELETE CASCADE,
    source_file_id INTEGER NOT NULL REFERENCES files(id),
    source_path TEXT NOT NULL,
    source_size_bytes INTEGER NOT NULL,
    source_partial_hash TEXT,
    target_drive_id INTEGER REFERENCES drives(id),
    target_drive_name TEXT,
    target_path TEXT,
    action TEXT NOT NULL,  -- 'copy_and_delete' or 'delete_only'
    status TEXT NOT NULL DEFAULT 'pending',  -- pending, copying, verifying, verified, deleted, failed, skipped
    error TEXT,
    started_at TEXT,
    completed_at TEXT
);

CREATE INDEX IF NOT EXISTS idx_migration_files_plan_id ON migration_files(plan_id);
CREATE INDEX IF NOT EXISTS idx_migration_files_status ON migration_files(plan_id, status);
```

Create `src/drivecatalog/migration.py` with four functions. All take a `sqlite3.Connection` as first argument (same pattern as consolidation.py).

**Function 1: `generate_migration_plan(conn, source_drive_name: str) -> dict`**

1. Call `get_consolidation_strategy(conn, source_drive_name)` to get the optimal file assignments. Re-raise ValueError if drive not found.
2. Get ALL files on the source drive (not just unique): `SELECT id, path, size_bytes, partial_hash FROM files WHERE drive_id = ?`
3. Build a set of unique file paths from the strategy's assignments (these need copy_and_delete).
4. For files NOT in the unique set (i.e., duplicated files that exist on other drives): action = 'delete_only', no target drive needed.
5. Insert a row into migration_plans with source_drive_id, source_drive_name, status='draft', computed totals.
6. Insert rows into migration_files for EVERY file on the source drive:
   - For unique files: action='copy_and_delete', target_drive_name and target_path from strategy assignments. Target path = same relative path as source. Look up target_drive_id from drives table by name.
   - For duplicated files: action='delete_only', target fields NULL.
7. Update the plan row with final counts (total_files, files_to_copy, files_to_delete, total_bytes_to_transfer).
8. conn.commit()
9. Return dict: `{"plan_id": int, "source_drive": str, "status": "draft", "total_files": int, "files_to_copy": int, "files_to_delete": int, "total_bytes_to_transfer": int, "is_feasible": bool}`. The is_feasible flag comes from the strategy result.

**Function 2: `validate_plan(conn, plan_id: int) -> dict`**

1. Load the plan from migration_plans. Raise ValueError if not found or status != 'draft'.
2. For each target drive used in the plan's migration_files (GROUP BY target_drive_id WHERE action='copy_and_delete'):
   - Sum the bytes to copy to that target.
   - Get the target drive's current free space: `SELECT total_bytes, used_bytes FROM drives WHERE id = ?`
   - Compare: if sum > free_bytes for any target, validation fails.
3. If all targets have sufficient space: UPDATE migration_plans SET status = 'validated'. conn.commit().
4. Return dict: `{"plan_id": int, "status": "validated" or "draft", "valid": bool, "target_space": [{"drive_name": str, "bytes_needed": int, "bytes_available": int, "sufficient": bool}]}`.
   If invalid, include which targets are insufficient.

**Function 3: `get_plan_details(conn, plan_id: int) -> dict | None`**

1. SELECT from migration_plans WHERE id = plan_id. Return None if not found.
2. Return dict with all plan columns plus a summary of per-status file counts: `SELECT status, COUNT(*), SUM(source_size_bytes) FROM migration_files WHERE plan_id = ? GROUP BY status`.
3. Dict keys: plan_id, source_drive_name, status, total_files, files_to_copy, files_to_delete, total_bytes_to_transfer, files_completed, bytes_transferred, files_failed, errors (parsed from JSON or empty list), operation_id, created_at, started_at, completed_at, file_status_counts (dict of status -> {"count": int, "bytes": int}).

**Function 4: `get_plan_files(conn, plan_id: int, status_filter: str | None = None, limit: int = 100, offset: int = 0) -> dict`**

1. Verify plan exists, raise ValueError if not.
2. Query migration_files with optional status filter, ordered by id, with limit/offset.
3. Return dict: `{"plan_id": int, "files": [{"id": int, "source_path": str, "source_size_bytes": int, "target_drive_name": str | None, "target_path": str | None, "action": str, "status": str, "error": str | None}], "total": int}` where total is the unfiltered count for the plan (or filtered count if status_filter given).

Import get_consolidation_strategy from drivecatalog.consolidation at the top.
