Add `execute_migration_plan(conn, plan_id: int, operation_id: str) -> dict` to migration.py. This function is designed to run synchronously in a background thread (same as _run_hash, _run_scan in drives.py). The operation_id links to the in-memory operation tracker for progress/cancellation.

**Imports to add at top of migration.py:**
- `import json, shutil` (shutil for file copy, json for errors list)
- `from pathlib import Path`
- `from datetime import datetime`
- `from drivecatalog.hasher import compute_partial_hash`
- `from drivecatalog.api.operations import is_cancelled, update_progress, update_operation, OperationStatus`

**Implementation steps:**

1. **Load and validate plan:**
   - SELECT from migration_plans WHERE id = plan_id.
   - Raise ValueError if not found.
   - Raise ValueError if status != 'validated' (only validated plans can execute).
   - UPDATE migration_plans SET status = 'executing', started_at = datetime('now'), operation_id = ?. conn.commit().
   - Also update the in-memory operation: `update_operation(operation_id, status=OperationStatus.RUNNING, started_at=datetime.now())`

2. **Load source and target drive mount paths:**
   - Source drive: `SELECT mount_path FROM drives WHERE id = ?` (source_drive_id from plan).
   - Raise ValueError if source drive not mounted (mount_path is NULL or path doesn't exist).
   - Build a dict of target_drive_id -> mount_path for all target drives in the plan: `SELECT DISTINCT target_drive_id FROM migration_files WHERE plan_id = ? AND target_drive_id IS NOT NULL`, then look up each.
   - Raise ValueError if any target drive is not mounted.

3. **Load all pending files:**
   - `SELECT * FROM migration_files WHERE plan_id = ? AND status = 'pending' ORDER BY id`
   - Store total count for progress tracking.

4. **Process files in a loop:**

   Initialize counters: files_completed = 0, bytes_transferred = 0, errors = []

   For each migration_file row:

   a. **Check cancellation:** Call `is_cancelled(operation_id)`. If true:
      - UPDATE migration_plans SET status = 'cancelled', completed_at = datetime('now'). conn.commit().
      - `update_operation(operation_id, status=OperationStatus.CANCELLED, completed_at=datetime.now())`
      - Return summary dict with what was completed so far.

   b. **If action == 'delete_only':**
      - The file is duplicated elsewhere. Just delete the source.
      - Build source_full_path = Path(source_mount_path) / row["source_path"]
      - UPDATE migration_files SET status = 'verified' (skip copy/verify since it's already backed up), then:
      - Try `source_full_path.unlink()`. If success: UPDATE status = 'deleted'. If OSError: UPDATE status = 'failed', error = str(e).
      - files_completed += 1

   c. **If action == 'copy_and_delete':**
      - Build source_full_path and target_full_path from mount paths + paths.
      - UPDATE migration_files SET status = 'copying', started_at = datetime('now'). conn.commit().

      **Copy with retry (max 2 attempts):**

      For attempt in range(2):
        - Create parent dirs: target_full_path.parent.mkdir(parents=True, exist_ok=True)
        - Use shutil.copy2(source_full_path, target_full_path) for the copy (preserves metadata, simpler than streaming for this use case).
        - On OSError: if attempt == 0, continue (retry). If attempt == 1, mark failed.

      If copy succeeded:
        - UPDATE migration_files SET status = 'verifying'. conn.commit().
        - Compute partial hash of destination: `compute_partial_hash(target_full_path, row["source_size_bytes"])`
        - Compare with row["source_partial_hash"].
        - **If source has no hash (source_partial_hash is NULL):** Compute hash of source too, compare both freshly computed hashes.
        - If hashes match: UPDATE status = 'verified'. conn.commit().
          - Now delete source: `source_full_path.unlink()`
          - UPDATE status = 'deleted', completed_at = datetime('now'). conn.commit().
          - bytes_transferred += row["source_size_bytes"]
          - files_completed += 1
        - If hashes don't match:
          - Remove the bad copy: target_full_path.unlink(missing_ok=True)
          - UPDATE status = 'failed', error = 'Hash mismatch after copy'. conn.commit().
          - errors.append(f"{row['source_path']}: hash mismatch")

      If copy failed after retries:
        - UPDATE status = 'failed', error = str(exception). conn.commit().
        - errors.append(f"{row['source_path']}: {str(exception)}")

   d. **Update progress** every file (not every 10 -- migrations are slower, fewer files):
      - `update_progress(operation_id, files_completed, total_files)`
      - UPDATE migration_plans SET files_completed = ?, bytes_transferred = ?, files_failed = (SELECT COUNT(*) FROM migration_files WHERE plan_id = ? AND status = 'failed'). conn.commit().

5. **Finalize:**
   - Count final statuses: `SELECT status, COUNT(*) FROM migration_files WHERE plan_id = ? GROUP BY status`
   - Determine final plan status: 'completed' if no pending/copying/verifying files remain, even if some failed.
   - UPDATE migration_plans SET status = final_status, completed_at = datetime('now'), errors = json.dumps(errors), files_completed, bytes_transferred, files_failed. conn.commit().
   - `update_operation(operation_id, status=OperationStatus.COMPLETED, progress_percent=100.0, completed_at=datetime.now(), result=summary_dict)`

6. **Return summary dict:**
   ```python
   {
       "plan_id": plan_id,
       "status": final_status,
       "files_moved": files_completed,  # successfully deleted
       "bytes_transferred": bytes_transferred,
       "files_failed": len(errors),
       "errors": errors,
   }
   ```

**Error handling:** Wrap the entire function body in try/except. On unexpected exception:
- UPDATE migration_plans SET status = 'failed', errors = json.dumps([str(e)]), completed_at = datetime('now'). conn.commit().
- `update_operation(operation_id, status=OperationStatus.FAILED, error=str(e), completed_at=datetime.now())`
- Re-raise the exception (let the API layer's background task handler log it).

**Important design notes:**
- Use `conn.commit()` after each file status change (not batch). This ensures state survives crashes. The WAL journal mode in database.py handles write performance.
- The function opens its OWN connection internally (do NOT rely on the passed conn being from the API thread -- background tasks run in a separate thread). Accept conn as parameter for testability but document that callers should pass a fresh connection.
- Actually, to match the existing _run_hash pattern which creates its own connection: change the signature to `execute_migration_plan(plan_id: int, operation_id: str) -> dict` (no conn param). Open conn inside with get_connection(). This is consistent with how _run_hash and _run_scan work.
