**Response models:**

```python
class MigrationFileResponse(BaseModel):
    """A single file in a migration plan."""
    id: int
    source_path: str
    source_size_bytes: int
    target_drive_name: str | None
    target_path: str | None
    action: str  # 'copy_and_delete' or 'delete_only'
    status: str  # pending, copying, verifying, verified, deleted, failed, skipped
    error: str | None

class FileStatusCount(BaseModel):
    """Count and bytes for a specific file status."""
    count: int
    bytes: int

class MigrationPlanResponse(BaseModel):
    """Full migration plan details."""
    plan_id: int
    source_drive_name: str
    status: str
    total_files: int
    files_to_copy: int
    files_to_delete: int
    total_bytes_to_transfer: int
    files_completed: int
    bytes_transferred: int
    files_failed: int
    errors: list[str]
    operation_id: str | None
    created_at: str
    started_at: str | None
    completed_at: str | None
    file_status_counts: dict[str, FileStatusCount]

class MigrationPlanSummary(BaseModel):
    """Brief plan info returned after generation."""
    plan_id: int
    source_drive: str
    status: str
    total_files: int
    files_to_copy: int
    files_to_delete: int
    total_bytes_to_transfer: int
    is_feasible: bool

class ValidatePlanResponse(BaseModel):
    """Result of plan validation."""
    plan_id: int
    status: str
    valid: bool
    target_space: list[TargetSpaceInfo]

class TargetSpaceInfo(BaseModel):
    """Free space check for a target drive."""
    drive_name: str
    bytes_needed: int
    bytes_available: int
    sufficient: bool

class MigrationFilesResponse(BaseModel):
    """Paginated list of migration files."""
    plan_id: int
    files: list[MigrationFileResponse]
    total: int

class ExecuteResponse(BaseModel):
    """Response after starting migration execution."""
    plan_id: int
    operation_id: str
    status: str
    poll_url: str
```

Order the classes so that referenced types are defined before they are used (TargetSpaceInfo before ValidatePlanResponse, etc.).

**Create `src/drivecatalog/api/routes/migrations.py`** with `router = APIRouter(prefix="/migrations", tags=["migrations"])`.

Follow the exact patterns from consolidation.py routes (explicit dict-to-model mapping per project decision) and copy.py (background task pattern).

**Five endpoints:**

1. **POST /migrations/generate** (response_model=MigrationPlanSummary)
   - Accepts GeneratePlanRequest body (source_drive: str).
   - Open conn via get_connection(), try/finally close.
   - Call generate_migration_plan(conn, request.source_drive).
   - Catch ValueError -> HTTPException(404).
   - Map returned dict to MigrationPlanSummary model explicitly (not **kwargs).

2. **GET /migrations/{plan_id}** (response_model=MigrationPlanResponse)
   - Path param plan_id: int.
   - Call get_plan_details(conn, plan_id).
   - If None -> HTTPException(404, "Migration plan not found").
   - Map dict to MigrationPlanResponse. For file_status_counts, map each status to FileStatusCount model.

3. **POST /migrations/{plan_id}/validate** (response_model=ValidatePlanResponse)
   - Call validate_plan(conn, plan_id).
   - Catch ValueError -> HTTPException(404 or 400 depending on message: not found = 404, wrong status = 400).
   - Map dict to ValidatePlanResponse, mapping each target_space entry to TargetSpaceInfo.

4. **POST /migrations/{plan_id}/execute** (response_model=ExecuteResponse)
   - Import BackgroundTasks from FastAPI.
   - Open conn to verify plan exists and status == 'validated'. If not: 404 or 400.
   - Close conn before starting background task.
   - Create operation: `op = create_operation("migration", source_drive_name)`
   - `background_tasks.add_task(_run_migration, plan_id, op.id)`
   - Return ExecuteResponse with plan_id, op.id, "started", f"/operations/{op.id}".

   **Define `_run_migration(plan_id: int, operation_id: str)` as a module-level function** (same pattern as _run_hash in drives.py):
   - Call `execute_migration_plan(plan_id, operation_id)`.
   - Wrap in try/except: on exception, update_operation with FAILED status.

5. **GET /migrations/{plan_id}/files** (response_model=MigrationFilesResponse)
   - Query params: status (optional str), limit (int, default 100), offset (int, default 0).
   - Call get_plan_files(conn, plan_id, status_filter=status, limit=limit, offset=offset).
   - Catch ValueError -> HTTPException(404).
   - Map dict to MigrationFilesResponse, mapping each file to MigrationFileResponse.

6. **DELETE /migrations/{plan_id}** (cancel endpoint)
   - Load plan, verify it exists and status == 'executing'.
   - If not executing: 400 "Plan is not currently executing".
   - Get operation_id from plan row. If None: 400 "No operation to cancel".
   - Call cancel_operation(operation_id) from operations module.
   - Return {"status": "cancelling", "plan_id": plan_id}.
