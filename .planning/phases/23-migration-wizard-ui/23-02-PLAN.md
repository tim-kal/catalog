  <action>
Create a new SwiftUI view presented as a sheet that implements a step-by-step consolidation wizard. The view manages its own state through a WizardStep enum.

**WizardStep enum:**
```
enum WizardStep {
    case analyzing       // Loading candidates from API
    case selectDrive     // User picks a source drive to consolidate
    case reviewPlan      // Shows migration plan details, user confirms
    case executing       // Live progress during migration
    case completed       // Summary of results
}
```

**State variables:**
- `@State private var step: WizardStep = .analyzing`
- `@State private var candidates: [ConsolidationCandidate] = []`
- `@State private var selectedCandidate: ConsolidationCandidate?`
- `@State private var plan: MigrationPlanSummary?`
- `@State private var planDetails: MigrationPlanResponse?`
- `@State private var validationResult: ValidatePlanResponse?`
- `@State private var migrationFiles: [MigrationFileResponse] = []`
- `@State private var operation: OperationResponse?`
- `@State private var error: String?`
- `@State private var isLoading = false`
- `@State private var pollTimer: Timer?`
- `@Environment(\.dismiss) private var dismiss`

**Body structure:**
Use a VStack with a switch on `step` to show the appropriate content. Include a consistent header area showing the current step name and a close/cancel button.

**Step 1: .analyzing**
- On appear, call `APIService.shared.fetchConsolidationCandidates()`
- Show a ProgressView spinner while loading
- On success, transition to `.selectDrive`
- On error, show error message with retry button

**Step 2: .selectDrive**
- Title: "Select Drive to Consolidate"
- Show list of candidates with `isCandidate == true` highlighted
- For each candidate, show: drive name, total files, unique files count, unique size, whether it's consolidatable, and target drives with free space
- Non-candidate drives shown dimmed with explanation (unique files don't fit on other drives)
- If no candidates exist (`consolidatableCount == 0`), show "No drives can be consolidated" message with dismiss button
- When user selects a candidate drive:
  1. Set `isLoading = true`
  2. Call `APIService.shared.generateMigrationPlan(sourceDrive: candidate.driveName)`
  3. Then call `APIService.shared.fetchMigrationPlan(planId: plan.planId)` to get full details
  4. Then call `APIService.shared.fetchMigrationFiles(planId: plan.planId, limit: 200)` to get file list
  5. Transition to `.reviewPlan`

**Step 3: .reviewPlan**
- Title: "Review Migration Plan"
- Show plan summary: source drive, total files, files to copy vs delete-only, total bytes to transfer
- Show file breakdown grouped by target drive (from `planDetails.fileStatusCounts` or by iterating `migrationFiles`). For each target: drive name, file count, total bytes
- Show a scrollable list of individual files with: filename (last path component), size, target drive, action (copy_and_delete or delete_only)
- Two buttons at bottom:
  - "Cancel" -- dismisses the sheet
  - "Validate & Confirm" -- calls `APIService.shared.validateMigrationPlan(planId:)`, then on success (valid == true) calls `APIService.shared.executeMigrationPlan(planId:)`, then transitions to `.executing`
  - If validation fails (valid == false), show which target drives lack space using `targetSpace` array

**Step 4: .executing**
- Title: "Migration in Progress"
- Show operation progress: percent complete, files processed / total, ETA
- Use Timer-based polling (every 2 seconds) on `APIService.shared.fetchOperation(id:)` -- same pattern used by DriveCard for scan/hash operations
- Also poll `APIService.shared.fetchMigrationFiles(planId:, limit: 50)` to show recent file activity
- Show a scrollable list of recently processed files with status indicators:
  - pending: gray circle
  - copying/verifying: blue spinner
  - deleted (success): green checkmark
  - failed: red X with error text
- Show a "Cancel Migration" button that calls `APIService.shared.cancelMigration(planId:)`
- When operation completes (status == "completed" or "failed" or "cancelled"):
  - Stop polling timer
  - Fetch final plan details via `APIService.shared.fetchMigrationPlan(planId:)`
  - Transition to `.completed`

**Step 5: .completed**
- Title: "Migration Complete" (or "Migration Cancelled" / "Migration Failed")
- Show summary from final planDetails:
  - Files completed (filesCompleted)
  - Bytes transferred (bytesTransferred) -- format as human-readable (KB/MB/GB)
  - Files failed (filesFailed)
  - If errors exist, show them in a disclosure group
- "Done" button dismisses the sheet

**Byte formatting helper:**
Add a private static function or extension to format Int64 bytes as human-readable strings (use 1024-based: B, KB, MB, GB, TB). This is used throughout the wizard for size display.

**Timer polling pattern (match DriveCard):**
- Start timer in `.executing` step
- Poll every 2 seconds
- Stop timer when operation is no longer active
- Clean up timer in `.onDisappear`

**Sheet sizing:**
- `.frame(minWidth: 600, minHeight: 500)` for comfortable wizard display
- Use `.frame(maxWidth: .infinity, maxHeight: .infinity)` on ScrollView content

**Error handling:**
- Wrap all API calls in do/catch
- On error, set `self.error = error.localizedDescription`
- Show error inline with a "Retry" button where applicable
